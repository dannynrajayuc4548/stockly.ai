## Getting Started

1. **Fork** the repository on GitHub
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/stockly.git
   cd stockly
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Setup

### Prerequisites
- Python 3.8+
- pip

### Install dependencies

```bash
pip install -r requirements.txt
```

### Run the app locally

```bash
streamlit run "stock dashboard.py"
```

The app will be available at `http://localhost:8501`.

## Pull Request Guidelines

Before submitting a pull request, please make sure:

- [ ] Your branch is up to date with `main`
- [ ] The app runs without errors locally
- [ ] Your changes are focused and minimal — one feature or fix per PR
- [ ] You've added a clear description of what changed and why
- [ ] Screenshots are included if you made any UI changes

### Submitting your PR

```bash
# Push your branch
git push origin feature/your-feature-name
```

Then open a Pull Request on GitHub against the `main` branch. Fill in the PR template with a clear title and description.

## Reporting Bugs

Found a bug? Please [open an issue](https://github.com/anshk1234/stockly/issues/new) and include:

- A clear title and description
- Steps to reproduce the bug
- Expected vs. actual behavior
- Screenshots if applicable
- Your Python and Streamlit versions (`streamlit --version`)

## Suggesting Features

Have an idea? [Open a feature request](https://github.com/anshk1234/stockly/issues/new) and describe:

- What the feature does
- Why it would be useful
- Any examples or references

## Code Style

- Follow standard **PEP 8** Python style conventions
- Use clear, descriptive variable and function names
- Keep Streamlit components organized — group related UI elements together
- Add comments for any logic that isn't immediately obvious

---

## Questions?

Feel free to open an issue if you're unsure about anything. All contributions, big or small, are genuinely valued. ⭐
